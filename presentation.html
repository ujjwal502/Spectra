Feature: User Authentication API
  Secure user login and profile management

  Scenario: Successful user login with valid credentials
    Given the system has a registered user with email "user@test.com"
    When I send a "POST" request to "/api/auth/login" with valid credentials
    Then I should receive a response with status code 200
    And the response should contain a valid JWT token
    And the token should have appropriate expiration time

  Scenario: Failed login with invalid credentials
    Given the system has security policies enabled
    When I send a "POST" request to "/api/auth/login" with invalid credentials
    Then I should receive a response with status code 401
    And the response should contain an error message about invalid credentials

  Scenario: Load testing user authentication endpoint
    Given the system is under normal load
    When I send 100 concurrent "POST" requests to "/api/auth/login"
    Then the average response time should be less than 500ms
    And no requests should fail due to server errors
